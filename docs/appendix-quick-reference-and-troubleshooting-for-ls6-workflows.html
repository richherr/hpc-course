Chapter 9 Appendix: Quick Reference and Troubleshooting for LS6 Workflows

9.1 Overview

This appendix is a compact reference for the commands, patterns, and reminders used throughout the course.

It is not a replacement for the explanatory chapters. Instead, it is a quick recovery tool: when you already understand the ideas, this appendix helps you remember the exact command pattern or troubleshooting sequence.

9.2 Core workflow picture

Keep this mental model in view:

Laptop -> Login node -> SLURM -> Compute node(s) -> Output

And keep this rule visible:

Do setup on the login node. Do serious computation on compute nodes.

9.3 A safe beginner shell pattern

Before doing something important, run:

pwd
ls

This quickly confirms:

  • where you are
  • what files are present
  • whether you are about to act in the intended directory

That tiny habit prevents many avoidable mistakes.

9.4 SSH

Connect to LS6:

ssh your_username@ls6.tacc.utexas.edu

9.5 Filesystem basics

Check current directory:

pwd

List files:

ls
ls -l

Change directories:

cd
cd ..
cd my_directory

Create directories:

mkdir project
mkdir -p project/src project/jobs project/output

Copy, move, rename, remove:

cp oldfile newfile
mv oldname newname
rm file

9.6 Editing files

Use a simple editor:

nano myfile.txt

9.7 Permissions

Make a script executable:

chmod +x script.sh

Inspect permissions:

ls -l script.sh

A common symptom of permission trouble is:

  • the file exists
  • but it will not run

9.8 Environment variables

View the environment:

env

Check common variables:

echo $HOME
echo $PATH
echo $WORK
echo $SCRATCH

Temporarily add a directory to PATH:

export PATH=$HOME/bin:$PATH

9.9 Modules

List available modules:

module avail

Load a module:

module load gcc

Inspect loaded modules:

module list

Unload a module:

module unload gcc

Check where a command comes from:

which gcc

9.10 Compilers and build commands

Basic C compile:

gcc hello.c -o hello

Compile with warnings and optimization:

gcc -Wall -O2 hello.c -o hello

Basic C++ compile:

g++ hello.cpp -o hello_cpp

Compile an OpenMP program:

gcc -fopenmp hello_omp.c -o hello_omp

Compile an MPI program:

mpicc hello_mpi.c -o hello_mpi

Build with make:

make

Clean with make clean if the Makefile provides it.

9.11 SLURM basics

Submit a job:

sbatch job.slurm

Monitor current jobs:

squeue -u $USER

Inspect job history:

sacct

Inspect one job:

sacct -j 123456

Cancel a job:

scancel 123456

9.12 Common job states

  • PD = pending
  • R = running
  • CG = completing
  • CD = completed

Remember:

  • PD does not automatically mean broken
  • sbatch does not mean “running right now”

9.13 Minimal serial batch script template

#!/bin/bash
#SBATCH -J my_job
#SBATCH -N 1
#SBATCH -n 1
#SBATCH -t 00:05:00
#SBATCH -p normal
#SBATCH -o my_job_%j.out
#SBATCH -e my_job_%j.err

pwd
hostname
date

9.14 Minimal MPI batch script template

#!/bin/bash
#SBATCH -J mpi_job
#SBATCH -N 1
#SBATCH -n 4
#SBATCH -t 00:05:00
#SBATCH -p normal
#SBATCH -o mpi_%j.out
#SBATCH -e mpi_%j.err

module load openmpi
mpirun ./hello_mpi

9.15 Minimal OpenMP batch script template

#!/bin/bash
#SBATCH -J omp_job
#SBATCH -N 1
#SBATCH -n 1
#SBATCH -c 4
#SBATCH -t 00:05:00
#SBATCH -p normal
#SBATCH -o omp_%j.out
#SBATCH -e omp_%j.err

export OMP_NUM_THREADS=4
./hello_omp

9.16 Output files

Inspect output:

cat slurm-*.out

Inspect a custom output file:

cat my_job_123456.out
cat my_job_123456.err

Use less when output is long:

less my_job_123456.out

9.17 Container commands

Check Apptainer:

which apptainer

Load it if necessary:

module load apptainer

Pull an image:

apptainer pull ubuntu.sif docker://ubuntu

Execute a command inside the image:

apptainer exec ubuntu.sif echo "hello"

Start an interactive shell:

apptainer shell ubuntu.sif

Bind the working directory:

apptainer exec --bind $PWD ubuntu.sif ls

9.18 Quick mental model table

Situation Good first question
command does not run Is the right module loaded?
job fails What do the output and error files say?
build fails What is the first real compiler error?
MPI output looks wrong Does the task count match the program model?
OpenMP uses one thread Did I set OMP_NUM_THREADS?
container cannot find files Did I bind the right host directory?

9.19 Troubleshooting checklist: shell and files

Use when something feels wrong immediately at the command line.

pwd
ls
which command_name
module list

Questions: - am I in the right directory? - is the file really present? - is the command actually available? - is the right environment loaded?

9.20 Troubleshooting checklist: build failure

When compilation fails:

  1. read the first meaningful error
  2. note the file and line number
  3. inspect nearby source lines
  4. fix one issue at a time
  5. recompile

Helpful commands:

ls
gcc -Wall hello.c -o hello
make

Useful questions: - is the file name correct? - did I forget a semicolon or brace? - did I forget a source file in the build? - did I load the compiler module?

9.21 Troubleshooting checklist: SLURM job failure

When a job fails or produces no useful result:

  1. check whether it is still in the queue
  2. inspect job accounting
  3. read output and error files
  4. verify current directory and input files
  5. check software environment assumptions

Helpful commands:

squeue -u $USER
sacct -j 123456
pwd
ls
cat my_job_123456.out
cat my_job_123456.err

Useful questions: - did the job actually start? - did the program inside the job fail? - am I using the right path? - is the right module loaded in the script?

9.22 Troubleshooting checklist: MPI and OpenMP

For MPI: - did I compile with mpicc? - did I request the intended number of tasks? - does the launch command match the code?

For OpenMP: - did I compile with -fopenmp? - did I request enough cores? - did I set OMP_NUM_THREADS?

9.23 Troubleshooting checklist: containers

When container behavior is confusing:

which apptainer
ls *.sif
apptainer exec image.sif which python3
apptainer exec --bind $PWD image.sif ls

Useful questions: - is Apptainer available? - does the image exist? - does the image contain the tool I need? - did I bind the directory containing my files?

9.24 Course workflow checklist

Before running a real job, confirm:

  • I am in the intended directory
  • the needed files exist
  • the required module(s) are loaded
  • the build step completed successfully
  • the SLURM request matches the workload
  • the output path is clear
  • I know what success should look like

9.25 Capstone checklist

Before submitting the capstone project, confirm:

  • the project directory is organized
  • the README explains build and run steps
  • the output files support the claims
  • the resource request is justified
  • at least one debugging reflection is included
  • another student could reproduce the workflow

9.26 Final reminder

Use the explanatory chapters to understand the ideas.

Use this appendix when you already know the idea and need the command pattern, checklist, or first troubleshooting step quickly.
