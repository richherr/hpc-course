Chapter 2 Introduction to High-Performance Computing (HPC)

2.1 Overview

High-performance computing (HPC) allows us to solve problems that are too large, too slow, or too complex for an ordinary desktop or laptop computer.

In this course, we will use the Lonestar6 (LS6) system at TACC as our working environment.

We will move from:

  • basic system access
  • to job submission
  • to compiling and running software
  • to modern workflows using containers

2.2 What is HPC?

At a high level, HPC is about:

  • More compute power
  • More memory
  • Parallel execution

Instead of one machine, we use a cluster.

2.3 Mental Model: The Cluster

Think of an HPC system as a collection of computers working together.

2.3.1 Key Components

  • Node → a single computer in the cluster
  • Core → a CPU processing unit within a node
  • Memory (RAM) → typically shared by processes on a node
  • Interconnect → the high-speed network connecting nodes

2.4 Mental Model: Login Nodes vs Compute Nodes

This is one of the most important distinctions in HPC.

2.4.1 Login Nodes

Used for:

  • editing files
  • compiling code
  • loading modules
  • moving data
  • submitting jobs

Not used for:

  • heavy computation
  • long-running analyses
  • large training runs

2.4.2 Compute Nodes

Used for:

  • actual program execution
  • scheduled workloads
  • parallel jobs
  • batch jobs
  • interactive compute sessions

⚠️ Important Rule

Serious computation should not be run on the login nodes.

2.5 Lonestar6 in Practice

On Lonestar6, users typically connect with SSH, work in a Linux shell, manage files across storage areas, and submit workloads to SLURM.

2.6 Connecting to the System

Use SSH to connect to LS6.

ssh your_username@ls6.tacc.utexas.edu

2.7 First Prompt

You may see something like:

[username@login1 ~]$

This tells you:

  • who you are
  • which machine you are on
  • your current directory

2.8 Storage Areas

You will often work with several storage areas:

  • $HOME → personal home directory
  • $WORK → project-oriented work storage
  • $SCRATCH → high-performance temporary storage

2.8.1 Check where you are

pwd

2.8.2 List files

ls

2.9 Basic Linux Commands

2.9.1 Move around

cd my_directory
cd ..
cd ~

2.9.2 Create a directory

mkdir project1

2.9.3 Copy a file

cp file1.txt file2.txt

2.9.4 Move or rename a file

mv oldname.txt newname.txt

2.9.5 Remove a file

rm file1.txt

2.10 Editing Files

Two common editors are:

nano myfile.txt

and

vim myfile.txt

2.11 Transfer Files

From your local system:

scp myfile.txt your_username@ls6.tacc.utexas.edu:~

2.12 First Small Script

Create a file:

nano hello.sh

Add:

echo "Hello from LS6"

Make it executable:

chmod +x hello.sh

Run it:

./hello.sh

2.13 Mental Model: What Just Happened?

You:

  • created a script
  • changed its permissions
  • ran a small command safely on a login node

That is appropriate because the task is tiny and brief.

2.14 Common Mistakes

  • running large jobs directly on the login node
  • forgetting chmod +x
  • losing temporary files in scratch storage
  • forgetting to check current location with pwd

2.15 Lab 1 — Getting Started on LS6

2.15.1 Part 1: Login

  1. Connect via SSH
  2. Confirm identity:

2.15.2 Part 2: Filesystem

pwd

2.15.3 Part 3: File Creation

nano hello.sh

echo "Hello HPC"

2.15.4 Part 4: Run It

chmod +x hello.sh
./hello.sh

2.16 Stretch Goal

Create a second script that prints:

  • your username
  • current directory
  • current date

Hint:

whoami
pwd
date

2.17 Looking Ahead

In the next chapter, we will deepen Linux skills by focusing on:

  • permissions
  • environment variables
  • modules
  • debugging habits

2.18 Key Takeaways

  • HPC systems use clusters
  • Login nodes are for setup and lightweight work
  • Compute nodes are for actual computation
  • Linux command-line literacy is essential
  • Storage choices matter
