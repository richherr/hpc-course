High-Performance Computing on Lonestar6

Chapter 3 Linux Fundamentals for HPC

3.1 Overview

In this chapter, we develop the Linux skills needed to work effectively on an HPC system.

You will learn:

  • how files and permissions work
  • how to inspect and manage your environment
  • how software is exposed through modules
  • how to think like a debugger on a shared system

3.2 Mental Model: You Are in a Shared Environment

Unlike a personal laptop, an HPC system is:

  • multi-user
  • resource-managed
  • environment-driven
  • structured for reproducibility

Every action you take should be:

  • deliberate
  • traceable
  • repeatable

3.3 The Linux Filesystem

Let’s visualize where you are:

/
├── home/
│   └── your_username/
├── work/
│   └── project_space/
├── scratch/
│   └── temporary_files/

3.3.1 Key Idea

  • /home → safe, small
  • /work → project data
  • /scratch → fast, temporary

3.4 Inspecting the System

3.4.1 Where am I?

pwd

3.4.2 What’s here?

ls -l

3.4.3 What does -l show?

-rw-r--r-- 1 user group size date filename

3.5 File Permissions

3.5.1 Mental Model

Every file has:

  • Owner
  • Group
  • Others

Each has:

  • read (r)
  • write (w)
  • execute (x)

3.6 Visual Breakdown

-rwxr-xr--
│││││││││
│││││││└── others
│││││└──── group
││└──────── owner

3.7 Changing Permissions

3.7.1 Make a file executable

chmod +x script.sh

3.7.2 Remove write permission

chmod -w file.txt

3.7.3 Numeric mode

chmod 755 script.sh

Meaning:

  • user: rwx
  • group: r-x
  • others: r-x

⚠️ Common HPC Failure

Many jobs fail because a script is missing execute permission.

3.8 Why Permissions Matter in HPC

Common failure modes include:

  • a script that will not run
  • a file you cannot edit
  • a shared file with the wrong access settings

3.9 Environment Variables

3.9.1 What are they?

Variables that define your working environment.

3.9.2 View them

env

3.9.3 Common examples

echo $HOME
echo $PATH

3.10 Mental Model: PATH

PATH tells the system where to look for executables.

echo $PATH

If a program is not in PATH, Linux may not find it.

3.10.1 Add to PATH temporarily

export PATH=$HOME/bin:$PATH

💡 Debugging Tip

If you see command not found, check:

echo $PATH
which <command>

3.11 Modules System

HPC systems do not expose all software globally.

Instead, they use modules.

3.11.1 Mental Model

Modules are loadable software environments.

3.12 Basic Module Commands

3.12.1 List available modules

module avail

3.12.2 Load a module

module load gcc

3.12.3 Check loaded modules

module list

3.12.4 Unload a module

module unload gcc

🧠 Key Insight

On HPC systems, software access is often environment-based, not system-wide in the way many users expect on a laptop.

3.13 Example: Using a Compiler

module load gcc
which gcc
gcc --version

3.14 Finding Files

find . -name "file.txt"

3.15 Viewing File Contents

cat file.txt
less file.txt
head file.txt
tail file.txt

3.16 Combining Commands with Pipes

cat file.txt | grep "error"

3.16.1 Pipe Mental Model

Think:

one command produces output, the next command filters or transforms it

3.17 Disk Usage

3.17.1 Check filesystem space

df -h

3.17.2 Check directory sizes

du -sh *

3.18 Processes

3.18.1 See your processes

ps -u $USER

3.18.2 Kill a process

kill <pid>

3.19 Optional R Integration

You can inspect environment settings inside R.

Sys.getenv("PATH")

You can also run shell commands from R.

system("echo $HOME")

3.20 Common Mistakes

  • forgetting chmod +x
  • PATH confusion
  • not loading needed modules
  • filling up $HOME
  • losing temporary work in $SCRATCH

3.21 Debugging Mindset in Linux

When something goes wrong, ask:

3.21.1 1. Where am I?

pwd

3.21.2 2. What files are here?

ls -l

3.21.3 3. Do I have permission?

ls -l script.sh

3.21.4 4. Is the program available?

which gcc

3.21.5 5. Is my environment correct?

module list

3.22.1 Part 1: Permissions

  1. Create a file:

  1. Check permissions:

  1. Make executable:

3.22.2 Part 2: Environment Variables

echo $HOME

export COURSE=HPC
echo $COURSE

3.22.3 Part 3: Modules

module avail

3.22.4 Part 4: Disk Usage

3.22.5 Part 5: File Search

touch example.txt

3.23 Stretch Goal

  • Create a custom bin/ directory
  • Add it to PATH
  • Put a script inside it
  • Run it without using the full path

3.24 Transition to HPC Workflows

So far, you have been:

  • navigating the system
  • editing files
  • running small commands

But on an HPC system, real work is usually not done directly from the command line on the login node.

Instead, you will:

  1. write a job script
  2. request resources
  3. submit the work to the scheduler

3.25 Looking Ahead

In the next chapter, we introduce SLURM, the scheduler that controls computation on the system.

3.26 Key Takeaways

  • Permissions control execution and modification
  • PATH controls command discovery
  • Modules manage software environments
  • Disk and process awareness matter
  • Reproducible HPC work depends on environment awareness
